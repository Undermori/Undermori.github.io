GitHub+PicGo搭建图床以及Typora自动上传设置

发表于2024-08-15|更新于2024-09-04|经验分享
|字数总计:360|阅读时长:1分钟|阅读量:

前言

使用GitHub搭建的图床相较于其他图床优点在于免费、限制少，缺点是国内IP的访问时不时会抽风，我因为没有网络连接问题与太大的图片需求所以目前足够满足我的需要，具体选择还是根据自己的实际情况决定。

GitHub相关配置

  1. 浏览器登陆自己的GitHub账户，点击页面顶部右上角的“+”，然后选择“New repository”。

  2. 在新页面为仓库命名并勾选“Public”，点击“Create repository”创建仓库。

    和GitHub Pages会要求仓库的命名不一样，仓库、分支、文件夹的命名只要自己可以看明白即可。

  3. 点击右上角的头像，选择“Settings”，在新页面的左侧边栏点击“Developer settings”后依次操作“Personal Access tokens”-“Tokens(classic)”-“Generate New Token(classic)”后显示为该界面：

  4. 在“Note”栏备注，“Expiration”栏选择token的有效期，勾选“repo”后“Generate token”。

  5. 复制并保存新页面的token。

PicGo相关配置

  1. 选择适合自己的版本下载安装：https://github.com/Molunerfinn/picgo/releases
  2. 打开PicGo，在“图床设置”中选择“GitHub”，按照自己刚才在GitHub中的配置填空。
  3. 如果这是你第一次用，也没有使用别的图床，记得再设置GitHub为默认图床。

Typora自动上传设置

在“文件”-“偏好设置”-“图像”中设置“上传服务”和你的PicGo路径。
